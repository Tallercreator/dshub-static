# DSHub — система компонентов UDF

UDF = сквозной нейминг от Figma до кода. Слои с префиксами, строго снизу вверх.
Референс: https://codetutorial.adc.ac/general/udf.html

## Слои
| Префикс | Слой | Что это | Примеры |
|---|---|---|---|
| `Q_` | квАрк | примитив / неделимый ассет (ниже атома) | Q_Icon, Q_Logo, Q_Illustration, Q_BrandMark, Q_BrandWordmark + токены |
| `A_` | атом | минимальный функциональный UI-элемент | A_Text, A_Button, A_Input, A_Checkbox, A_Tag, A_CategoryTag, A_SegmentItem, A_FilterChip |
| `M_` | молекула | группа атомов с одной функцией | M_Segment, M_CasePreview, M_FormField, M_CaseTags, M_TermChip |
| `O_` | организм | самостоятельный функциональный блок | O_Header, O_CaseCard, O_Form, O_Footer, O_FilterSidebar |
| `W_` | обёртка | layout-структура (сетки, контейнеры) | W_PageContainer, W_CardGrid, W_SectionHeader |
| `S_` | суперорганизм | секция страницы | S_Hero, S_CasesSection, S_FormSection |
| `T_` | шаблон | страница | T_Home, T_Cases, T_CasePage, T_TermPage |

## Правила
1. **Снизу вверх.** Не начинай компонент, пока не готовы его части. Организм собирается из инстансов готовых атомов/молекул, не «рисуется на месте».
2. **Сквозной нейминг.** Имя в Figma = имя компонента/класса в коде. Не переименовывать на хэндофе.
3. **Биндинг токенов.** Атомы/компоненты ссылаются только на **semantic**-токены, не на сырой core.
4. **КвАрк vs атом.** Глиф/ассет = квАрк (Q_Icon). Обёртка с пропами size/color = атом (A_Icon) — нужна в основном в коде; в Figma можно обойтись Q_Icon с вариантами.
5. **Состояния — это не пропсы.** В Figma состояния (Normal/Hover/Pressed/Disabled) рисуются как варианты, но в коде `hover`/`pressed` — это CSS (`:hover`, `:active`), пропсы — только смысловые (`value`, `disabled`, `variant`).
6. **Граница атом/молекула.** Голое поле = `A_Input` (атом). Поле + лейбл + ошибка = `M_FormField` (молекула). Не зашивай лейбл в атом — потеряешь переиспользование.
